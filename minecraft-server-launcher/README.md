# Minecraft Server Launcher (tmux)

Скрипт для запуска одного или нескольких **Minecraft‑серверов внутри tmux‑сессии**. Основная задача — централизованный старт серверов при запуске системы и удобное управление ими через одну tmux‑сессию.

## Назначение проекта

Проект предназначен для администраторов Minecraft‑серверов, у которых:

- используется один или несколько серверов;
- требуется запускать их в фоновом режиме;
- важно иметь возможность подключаться к консоли каждого сервера через tmux.

Скрипт создаёт tmux‑сессию, настраивает окна и панели и запускает серверы через их стандартные `start.sh` файлы.

---

## Контекст использования

Типовой сценарий:

- сервер перезагружается;
- при старте системы автоматически создаётся tmux‑сессия;
- внутри неё запускаются все необходимые Minecraft‑серверы;
- администратор в любой момент может подключиться к сессии и наблюдать за работой серверов.

Это особенно удобно при размещении нескольких серверов на одной машине.

---

## Принцип работы

Скрипт выполняет последовательность команд tmux:

1. Создаёт новую tmux‑сессию.
2. Переименовывает окна (по версиям или назначению серверов).
3. Делит окна на панели при необходимости.
4. Отправляет команды запуска серверов (`./start.sh`) в соответствующие панели.

В результате все серверы работают внутри одной tmux‑сессии и доступны для управления.

## Установка tmux

Если `tmux` ещё не установлен:

```bash
# Arch Linux
pacman -S tmux

# Debian / Ubuntu / Linux Mint
apt install tmux

# Fedora
dnf install tmux

# RHEL / CentOS
yum install tmux

# macOS (Homebrew)
brew install tmux

# macOS (MacPorts)
port install tmux

# openSUSE
zypper install tmux
```

## Запуск при старте системы

Для автоматического запуска серверов при старте системы используется `crontab` **от имени пользователя**, под которым запускаются сервера Minecraft.

Откройте crontab:

```bash
crontab -e
```

И добавьте строку:

```bash
@reboot /path/to/tmux_minecraft.sh

```

> В конце файла должна быть пустая строка, иначе `crontab` может выдать ошибку при сохранении.

## Настройка скрипта

Внутри скрипта настраиваются:

- имена tmux‑сессий;
- имена окон;
- пути к каталогам серверов;
- команды запуска (`./start.sh`).

Пример базовых команд tmux:

```bash
tmux new -s MCS -d
tmux rename-window Bungee_Lobby
tmux send-keys -t MCS:1.1 'cd /server/BungeeCord && ./start.sh' Enter
tmux split-window -h -t MCS:1.1
tmux send-keys -t MCS:1.2 'cd /server/Lobby && ./start.sh' Enter
```

Нумерация окон может быть изменена в `~/.tmux.conf`:

```bash
set -g base-index 1
set -g history-limit 10000
```

## Пример упрощённого сценария

Для одного сервера возможен минимальный вариант:

```bash
tmux new -s MCS -c /server/GTNH274/ -d
tmux send-keys -t MCS:1 './start.sh' Enter
```
